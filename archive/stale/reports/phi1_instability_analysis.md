## Executive Root Cause
Phi1 instability is not a single numerical fluctuation. It is caused by mixing two non-equivalent Phi1 pipelines:
1. Canonical hermetic reconstruction pipeline (stable, improves reconstruction at all temperatures).
2. Local/intersection Phi1 pipeline (coarse-grid, partial-overlap representation that is only partially aligned to legacy/canonical shape and degrades reconstruction).

This is why the system can simultaneously show "Phi1 appears stable" in one artifact and "Phi1 does not improve reconstruction" in another.

## 4) Pipeline Sensitivity Differences
Key differences causing non-transferability:
- Normalization method:
  - non_canonical_phi1 uses S_norm and mean local backbone subtraction (reports/kappa1_phi1_local_v2_20260329_234420.md).
  - Canonical reconstruction uses strict LOTO residual construction with holdout exclusion.
- T subsets:
  - Canonical uses 14 temperatures (4-30 K).
  - reconstruction_v1 uses N_T intersection=9 only.
- Sampling density:
  - reconstruction_v1 uses N_I intersection=6.
  - Canonical uses full canonical reconstruction grid.
- Windowing/overlap:
  - representation bridge has unmatched local temperatures: 28,30,32,34 K.
  - normalized scale mismatch = 3.521.
  - SAFE_TO_PROCEED_TO_RECONSTRUCTION = NO.

## Concrete Root-Cause Statement
The instability comes from using a non_canonical_phi1 representation (coarse intersection grid + different normalization + partial T overlap) as if it were canonically equivalent. It is not shape-equivalent (cosine 0.696, explained variance 0.417), so when injected into reconstruction it systematically degrades RMSE. Regime effects (kappa2-linked) amplify the mismatch but are not the primary trigger; the trigger is pipeline non-equivalence plus normalization transfer failure.
