# Aging Plotting Guide

## Figure taxonomy

Observable-level figures and decomposition figures are different objects.

Warning:

These two figure types represent fundamentally different objects and must not be confused.

## A. Observable-level figures (BASIC)

These figures summarize one scalar value per pause temperature and are the correct basic outputs for physical interpretation.

- `DeltaM vs Temperature`
  - all pause runs on the common temperature grid
  - generated by `Main_Aging -> stage8_plotting -> plotAgingMemory(...)`
- `AFM-like vs T_pause` and `FM-like vs T_pause`
  - one scalar AFM-like value and one scalar FM-like value per pause temperature
  - generated by `Main_Aging -> stage6_extractMetrics(...)`
  - figure name: `Aging memory summary`

Exact current summary path:

- function: `stage6_extractMetrics`
- file: `Aging/pipeline/stage6_extractMetrics.m`
- call chain: `Main_Aging -> stage6_extractMetrics`

This is the figure family that `cfg.mode = 'basic_plots'` should use.

## B. Decomposition figures (DIAGNOSTIC)

These figures show the temperature-dependent decomposition of a single run into:

- `DeltaM(T)`
- `DeltaM_smooth(T)`
- `DeltaM_sharp(T)`

They are used for debugging and analysis of the decomposition procedure, not for the basic observable view.

Exact current diagnostic path:

- function: `plotAFM_FM_decomposition`
- file: `Aging/plots/plotAFM_FM_decomposition.m`
- call chain: `Main_Aging -> stage4_analyzeAFM_FM -> plotDecompositionExamples -> plotAFM_FM_decomposition`

## Keep Using As-Is

Use these existing figure producers as-is for basic requests:

- `plotAgingMemory(...)` for `DeltaM vs Temperature`
- `stage6_extractMetrics(...)` for `AFM-like / FM-like vs T_pause`

## Do Not Use For Basic Requests

Do not use these when the request is only for basic observable-level figures:

- `plotAFM_FM_decomposition(...)`
- `plotDecompositionExamples(...)`

They are diagnostic decomposition views of a single pause run.
