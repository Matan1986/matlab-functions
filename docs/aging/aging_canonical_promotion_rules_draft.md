# Aging canonical promotion rules (draft, F6Z)

**Status:** Draft — checklist rows in `tables/aging/aging_F6Z_canonical_promotion_rules.csv`.

## Separation

- **`canonical_candidate`**: staging after parity/review; not policy canonical until governance completes (`aging_namespace_contract.md`).
- **`canonical`**: ratified registry rows + governance metadata only.
- **`diagnostic`**: not promoted without new registry rows.
- **`legacy_quarantine`**: evidence-readable; not canonical inputs without migration (`aging_agent_assistive_enforcement.md`).

## Minimum themes

Observable definition id, registry_id, namespace, writer_id, formula_id, source_run_id, input lineage, sidecar/manifest, sign conventions, units/status, scalarization recipe for comparable dips, tau/R identities when derived, validator posture, no unresolved plain **`Dip_depth`** for model-ready paths, authoritative flags over misleading field names (`CP014`–`CP015`).
