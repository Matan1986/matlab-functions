# Output Artifacts

Use these rules for organizing analysis outputs inside every run directory.

Canonical run root:

`results/<experiment>/runs/run_<timestamp>_<label>/`

## Required artifact directories

Each run must contain these artifact directories:

- `figures/`
- `tables/`
- `reports/`
- `review/`

Agents must not invent new artifact directories such as `plots/`, `figs/`, or `analysis_outputs/`.

## Figures

All figures must be saved in `figures/`.

Examples:

- `figures/aging_map_heatmap.png`
- `figures/aging_map_heatmap.fig`

Store both presentation-ready and editable figure formats there.

## Tables

Numeric outputs must be stored in `tables/`.

Examples:

- `tables/observables.csv`
- `tables/fit_summary.csv`

Use `tables/` for CSV and similar machine-readable numeric outputs.

## Reports

Text summaries must be stored in `reports/`.

Examples:

- `reports/run_summary.md`
- `reports/interpretation_notes.txt`

Use `reports/` for human-readable written output generated by the analysis.

## Review

ZIP files prepared for human inspection must be stored in `review/`.

Examples:

- `review/aging_geometry_review.zip`
- `review/switching_alignment_package.zip`

Use `review/` only for review bundles and handoff archives.

## Naming rule

Keep filenames descriptive and analysis-specific so that artifacts remain understandable when viewed outside MATLAB.

## Agent rule

Agents must follow this artifact layout for every new run and must not create alternative folders for the same output types.
